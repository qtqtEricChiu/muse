## 问题 1：双语 LRC 解析逻辑错误

### 问题描述
当前 `parseLyricText()` 函数（app.js ~L1294）错误地将**上一句的翻译**和**下一句的原文**合并在一起显示。

### 正确的 LRC 格式（用户提供示例）
```
[00:02.69] Cross my heart          ← 第1句原文（时间戳 A）
[00:03.84] 胸前画十字 郑重起誓      ← 第1句中文翻译（时间戳 B）
[00:03.84] we'll always            ← 第2句原文（时间戳 B）
[00:04.70] 我们会永远              ← 第2句中文翻译（时间戳 C）
```

**关键规则**：
- 同时间戳的第一行 = 原文
- 同时间戳的第二行 = 翻译
- 显示时必须原文+翻译成对出现

### 修复方案

#### 步骤 1：重写 `parseLyricText()` 函数（app.js ~L1294）
```javascript
// 旧逻辑（错误）：两遍扫描合并同时间戳
// 新逻辑（正确）：按时间戳分组，同组第一行=original，第二行=translation

function parseLyricText(text) {
    const lines = text.split(/\r?\n/);
    const timeMap = new Map(); // time -> {original, translation}
    
    for (const line of lines) {
        const timeMatch = line.match(/\[(\d{2}):(\d{2})(\.(\d{2,3}))?\]/);
        if (!timeMatch) continue;
        
        const min = parseInt(timeMatch[1]);
        const sec = parseInt(timeMatch[2]);
        const ms = timeMatch[4] ? parseInt(timeMatch[4].padEnd(3, '0')) : 0;
        const time = min * 60 + sec + ms / 1000;
        
        const txt = decodeText(line.replace(/\[.*?\]/g, '').trim());
        if (!txt) continue;
        
        if (!timeMap.has(time)) {
            timeMap.set(time, { original: txt, translation: '' });
        } else {
            // 同时间戳第二行 = 翻译
            timeMap.get(time).translation = txt;
        }
    }
    
    // 转换为数组
    const result = [];
    for (const [time, data] of timeMap) {
        result.push({
            time,
            text: data.original, // 默认显示原文
            original: data.original,
            translation: data.translation,
            isBilingual: !!data.translation
        });
    }
    
    result.sort((a, b) => a.time - b.time);
    return result;
}
```

## 问题 4：Crossfade 交叉歌曲错误

### 问题描述
用户发现 crossfade（音轨交叉淡入淡出）功能有问题：
- **现象**：crossfade 交叉播放的是**另一首歌**，而不是播放列表中**准备播放的下一首**
- **结果**：crossfade 的歌曲与下一首实际播放的歌曲**同时播放**，产生混音

### 根因分析

#### Bug 定位：`triggerCrossfade()` 函数（app.js ~L1710）

**问题代码**：
```javascript
function triggerCrossfade() {
    const userVolume = parseFloat(el.volSlider.value);
    const nextIndex = isShuffle 
        ? Math.floor(Math.random() * playlist.length)  // 🚫 BUG: 随机选歌
        : (currentIndex + 1) % playlist.length;        // ✅ 正常顺序下一首
    
    // 创建第二个音频元素用于预加载下一首
    const nextAudio = new Audio();
    nextAudio.src = playlist[nextIndex].url;
    // ...
}
```

**核心问题**：
1. **Shuffle 模式下随机选择**：`Math.floor(Math.random() * playlist.length)` 随机选一首歌
2. **与 `goNext()` 逻辑不一致**：`goNext()` 也是随机选歌，但两次随机选择的结果**几乎不可能相同**
3. **时序冲突**：
   - `triggerCrossfade()` 开始 → 加载 Song B
   - Crossfade 进行中 → Song A 淡出，Song B 淡入
   - Song A 结束 → `onended` 触发 → `goNext()` 随机选了 Song C
   - **结果**：Song B（crossfade）和 Song C（`goNext()`）同时播放

#### 正确逻辑应该是：
**Crossfade 的下一首 = `goNext()` 将要播放的下一首**

### 修复方案

#### 步骤 1：统一下一首选择逻辑
新增辅助函数 `getNextTrackIndex()`（app.js ~L1650）：

```javascript
/**
 * 获取下一首要播放的歌曲索引
 * 与 goNext() 逻辑保持一致
 */
function getNextTrackIndex() {
    if (!playlist.length) return -1;
    
    if (isShuffle) {
        // Shuffle 模式：排除当前歌曲，随机选择一首
        if (playlist.length <= 1) return currentIndex;
        let next;
        do {
            next = Math.floor(Math.random() * playlist.length);
        } while (next === currentIndex);
        return next;
    } else {
        // 顺序模式：下一首
        return (currentIndex + 1) % playlist.length;
    }
}
```
