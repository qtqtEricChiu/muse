## 问题 1：主界面歌词栏垂直居中不准确

### 问题描述
当一行歌词因为空间问题多次换行时，或者歌曲刚开始时，主界面歌词栏可能存在定位不准确的问题，导致歌词并不是完全水平垂直居中。

### 根因分析

#### 根因 1：`clientHeight` 在 CSS 过渡期间读取
**代码位置:** `app.js` ~L1461-1468 (`syncLyrics` 函数)

**问题代码:**
```javascript
if (lyricsAlignMode === 'center') {
    offset = line.offsetTop - el.lrcView.offsetTop
        - (el.lrcView.clientHeight / 2) + (line.clientHeight / 2);
}
```

**问题分析:**
- `line.clientHeight` 在 CSS `transition: all 0.6s` 过渡期间读取
- 浏览器可能在过渡**中间状态**返回 `clientHeight`（例如从 40px → 60px 的过渡中返回 50px）
- 导致 `offset` 计算偏差，歌词"跳动"

**修复思路:**
- 在读取 `clientHeight` 前强制重排（`void line.offsetHeight`）
- 或使用 `getBoundingClientRect().height` 获取渲染后实际高度

---

#### 根因 2：`.lrc-viewport` 的 `padding: 50%` 导致 `offsetTop` 计算偏差
**代码位置:** `css/base-layout.css` ~L127

**问题代码:**
```css
.lrc-viewport { 
    flex: 1; 
    overflow-y: auto; 
    overflow-x: hidden;
    text-align: var(--lrc-align); 
    padding: 50% 20px;  /* 🚫 BUG: 50% 基于宽度，不是高度！ */
    scroll-behavior: smooth; 
    width: 100%;
}
```

**问题分析:**
- CSS 规范：`padding` 的百分比**基于父元素宽度**，不是高度
- 如果 `.lrc-viewport` 宽度是 450px，`padding: 50%` = 225px（上下各 225px）
- 当窗口高度变化时，垂直居中位置**不正确**
- `line.offsetTop` 是相对于 `.lrc-viewport` 的，已经包含了 `padding-top`
- 计算 `offset` 时又减去 `el.lrcView.offsetTop`（可能是 0 或错误值），导致双重偏差

**修复思路:**
- 移除 `padding: 50%`
- 改用 `padding: 20px`（固定值）
- 或在 JS 中动态计算 `padding-top`（基于 `.lrc-viewport` 实际高度）

#### 根因 3：`transform: scale(1.05)` 未纳入计算
**代码位置:** `css/base-layout.css` ~L147

**问题代码:**
```css
.lrc-line.active { 
    color: #fff; 
    font-size: calc(var(--lrc-font-size) * 1.44); 
    font-weight: 700; 
    text-shadow: 0 0 20px rgba(154,200,226,0.6); 
    transform: scale(1.05);  /* 🚫 BUG: 视觉高度 > 布局高度 */
    filter: blur(0px); 
    opacity: 1; 
}
```

**问题分析:**
- `transform: scale(1.05)` **只影响视觉渲染**，不影响 `clientHeight` / `offsetTop`
- 视觉高度 = `clientHeight * 1.05`
- 但 `line.clientHeight / 2` 计算的是**布局高度的一半**，不是视觉高度的一半
- 导致歌词"看起来"偏上 ~2.5%

**修复思路:**
- 在 `offset` 计算中纳入 `scale` 因子：
  ```javascript
  const scale = 1.05;
  offset = line.offsetTop - el.lrcView.offsetTop
      - (el.lrcView.clientHeight / 2) + (line.clientHeight * scale / 2);
  ```

#### 根因 4：未强制重排就读取布局属性
**代码位置:** `app.js` ~L1461-1468 (`syncLyrics` 函数)

**问题分析:**
- `syncLyrics()` 在 `timeupdate` 事件中触发（每秒多次）
- 浏览器可能**批量处理**布局重排（Lazy Reflow）
- 读取 `line.offsetTop` / `line.clientHeight` 时，可能获取到**旧值**
- 导致歌词"抖动"或"跳动"

**修复思路:**
- 在读取布局属性前强制重排：
  ```javascript
  // 强制重排（读取 offsetHeight 会触发重排）
  void line.offsetHeight;
  
  // 现在读取的值是准确的
  const lineTop = line.offsetTop;
  const lineHeight = line.clientHeight;
  ```

```javascript
// 🩹 v2.8.9: 修复垂直居中计算（4个根因）
const syncLyrics = (force = false) => {
    if(!parsedLyrics.length) return;
    const cur = audio.currentTime - lyricsOffset;
    let activeIdx = -1;
    for (let i = parsedLyrics.length - 1; i >= 0; i--) {
        if (cur >= parsedLyrics[i].time) { activeIdx = i; break; }
    }

    // 主界面歌词面板高亮
    if (el.lrcPanel.style.display !== 'none') {
        const lines = el.lrcView.querySelectorAll('.lrc-line');
        lines.forEach((line, i) => {
            if (i === activeIdx) {
                if (!line.classList.contains('active')) {
                    line.classList.add('active');
                    if (!isUserScrollingLyrics || force) {
                        // 🩹 根因4: 强制重排
                        void line.offsetHeight;
                        
                        // 🩹 根因1: 使用 getBoundingClientRect() 获取准确高度
                        const rect = line.getBoundingClientRect();
                        const viewRect = el.lrcView.getBoundingClientRect();
                        
                        let offset;
                        if (lyricsAlignMode === 'center') {
                            // 模式 A：完全垂直居中
                            // 🩹 根因3: 纳入 transform: scale(1.05)
                            const scale = 1.05;
                            const visualHeight = rect.height * scale;
                            offset = (rect.top - viewRect.top)
                                - (viewRect.height / 2) + (visualHeight / 2);
                        } else {
                            // 模式 B：偏上显示（距离顶部 30%）
                            offset = (rect.top - viewRect.top)
                                - (viewRect.height * 0.3);
                        }
                        
                        // 🩹 根因2: 移除 padding: 50% 后，offset 计算简化
                        // 不再需要减去 el.lrcView.offsetTop（因为 rect.top 已经是相对于视口）
                        el.lrcView.scrollTo({ top: offset, behavior: 'smooth' });
                    }
                }
            } else {
                line.classList.remove('active');
            }
        });
    }
    
    // ... 后续代码不变
};
```

**关键修改点:**
1. 使用 `getBoundingClientRect()` 替代 `offsetTop` / `clientHeight`
2. 纳入 `transform: scale(1.05)` 的影响
3. 强制重排（`void line.offsetHeight`）
4. 简化 `offset` 计算（不再需要 `el.lrcView.offsetTop`）

## 问题 2：修复双语歌词解析（翻译链算法增强）

### 问题描述
用户提供的 LRC 文件示例（`Just Keep Watching.lrc`）显示，末尾的翻译行时间戳与下一句原文时间戳相同，但翻译应该与**上一句的原文**配对，而不是与下一句原文配对。

**示例 LRC 结构:**
```
[02:17.63]Just keep watchin'        ← 原文1
[02:18.38]只管继续欣赏              ← 翻译1（对应原文1）
[02:18.38]As long as you watchin' Yeah ← 原文2（与翻译1同时间戳）
[02:20.94]只要你目光追随 没错        ← 翻译2（对应原文2）
[02:20.94]Just keep watchin'        ← 原文3（与翻译2同时间戳）
[02:25.09]只管继续欣赏              ← 翻译3（对应原文3）
```

**当前问题:**
- `parseLyricText()` 将 `[02:20.94]` 的两行都放入 `group.entries`
- 如果翻译行为空（或不存在），会将下一句原文错误地当作翻译

### 根因分析

**代码位置:** `app.js` ~L1307-1370 (`parseLyricText` 函数)

**问题代码:**
```javascript
// Step 2: 按时间戳分组（保持同时间戳内顺序）
const groups = [];
for (const entry of entries) {
    const last = groups[groups.length - 1];
    if (last && Math.abs(last.time - entry.time) < 0.001) {
        last.entries.push(entry.text);
    } else {
        groups.push({ time: entry.time, entries: [entry.text] });
    }
}

// Step 3: 链式构建歌词 — 翻译顺延到上一句
const result = [];
let pendingOriginal = null; // 等待翻译赋值的上一句原文

for (const group of groups) {
    const hasTranslation = group.entries.length > 1
        && group.entries[0]
        && group.entries[0].trim();

    const texts = group.entries.filter(t => t && t.trim());
    if (texts.length === 0) continue;

    // 🚫 BUG: 如果 hasTranslation=false，翻译会被忽略，但 pendingOriginal 仍然赋值
    if (pendingOriginal && result.length > 0 && hasTranslation) {
        result[result.length - 1].translation = texts[0];
        result[result.length - 1].isBilingual = true;
        result[result.length - 1].text =
            result[result.length - 1].original + '\n' + texts[0];
    }

    // 本组最后一行 = 新的原文
    const original = texts[texts.length - 1];
    result.push({
        time: group.time,
        text: original,
        original: original,
        translation: null,
        isBilingual: false
    });
    pendingOriginal = original;
}
```

**问题分析:**
1. **同时间戳分组错误**  
   - `[02:18.38]` 的两行被分到同一组：`entries = ["只管继续欣赏", "As long as you watchin' Yeah"]`
   - 但第一行是翻译，第二行是原文，不应该在同一组

2. **翻译配对逻辑错误**  
   - 当前逻辑：同时间戳第一行 = 翻译，最后一行 = 原文
   - 正确逻辑：翻译应该与**上一时间戳的原文**配对，不是本组原文

3. **空翻译处理不当**  
   - 如果某句歌词没有翻译（LRC 文件中缺少翻译行），`hasTranslation` 为 `false`
   - 但 `pendingOriginal` 仍然被赋值，导致下一句有翻译时，错误地配对了上上句原文

### 修复方案（3 步骤）

#### 步骤 1：修改 `parseLyricText()` 的分组逻辑
**修改文件:** `app.js`  
**修改函数:** `parseLyricText()` (~L1307-1370)

**修改代码:**
```javascript
// 🩹 v2.8.9: 修复双语歌词解析（翻译链算法增强）
function parseLyricText(text) {
    const lines = text.split(/\r?\n/);

    // Step 1: 解析所有带时间戳的条目（保持原始顺序）
    const entries = [];
    for (const line of lines) {
        const match = line.match(/\[(\d{2}):(\d{2})(\.(\d{2,3}))?\]/);
        if (!match) continue;
        const min = parseInt(match[1]);
        const sec = parseInt(match[2]);
        const ms = match[4] ? parseInt(match[4].padEnd(3, '0')) : 0;
        const time = min * 60 + sec + ms / 1000;
        const txt = decodeText(line.replace(/\[.*?\]/g, '').trim());
        entries.push({ time, text: txt });
    }

    // Step 2: 🩹 不再按时间戳分组，改为逐行处理（保留所有行）
    // 翻译行紧跟在原文行后面（可能同时间戳，也可能不同）
    const result = [];
    let pendingOriginal = null; // 等待翻译赋值的上一句原文
    let i = 0;

    while (i < entries.length) {
        const entry = entries[i];
        
        // 判断是否为制作信息行（非歌词正文）
        const isMeta = /(曲|编曲|词|混音|录音|制作|吉他|贝斯|键盘|鼓|和声|弦乐|配唱|OP|SP|ISRC|TME享有|版权)[：:]/.test(entry.text);
        if (isMeta) {
            i++;
            continue;
        }

        // 判断本行是否为翻译（通过启发式规则）
        const isTranslation = (text) => {
            // 规则1: 如果上一行是原文，本行是翻译（同时间戳或时间戳差 < 0.1s）
            if (pendingOriginal && Math.abs(entry.time - pendingOriginal.time) < 0.1) {
                return true;
            }
            // 规则2: 如果本行包含中文字符，且上一行不包含 → 可能是翻译
            const hasChinese = (t) => /[\u4e00-\u9fa5]/.test(t);
            if (pendingOriginal && hasChinese(entry.text) && !hasChinese(pendingOriginal.original)) {
                return true;
            }
            return false;
        };

        if (isTranslation(entry.text)) {
            // 本行是翻译 → 配对到上一句原文
            if (pendingOriginal && result.length > 0) {
                result[result.length - 1].translation = entry.text;
                result[result.length - 1].isBilingual = true;
                result[result.length - 1].text =
                    result[result.length - 1].original + '\n' + entry.text;
            }
            i++;
        } else {
            // 本行是原文 → 新增歌词条目
            result.push({
                time: entry.time,
                text: entry.text,
                original: entry.text,
                translation: null,
                isBilingual: false
            });
            pendingOriginal = result[result.length - 1];
            i++;
        }
    }

    return result;
}
```

**关键修改点:**
1. **移除按时间戳分组逻辑** — 改为逐行处理，保留所有行
2. **新增 `isTranslation()` 启发式规则** — 判断某行是否为翻译
3. **翻译配对到上一句原文** — 而不是本组原文

#### 步骤 2：（可选）增强翻译检测规则
**修改文件:** `app.js`  
**修改函数:** `parseLyricText()` 内的 `isTranslation()` 函数

**增强规则:**
```javascript
const isTranslation = (text) => {
    // 规则1: 如果上一行是原文，本行是翻译（同时间戳或时间戳差 < 0.1s）
    if (pendingOriginal && Math.abs(entry.time - pendingOriginal.time) < 0.1) {
        return true;
    }
    // 规则2: 如果本行包含中文字符，且上一行不包含 → 可能是翻译
    const hasChinese = (t) => /[\u4e00-\u9fa5]/.test(t);
    if (pendingOriginal && hasChinese(entry.text) && !hasChinese(pendingOriginal.original)) {
        return true;
    }
    // 🚀 规则3: 如果本行是纯中文，上一行是纯英文 → 可能是翻译
    const isPureChinese = (t) => /^[\u4e00-\u9fa5，。！？、；：""''（）【】《》]+$/.test(t);
    const isPureEnglish = (t) => /^[a-zA-Z0-9\s\.,!?'"-]+$/.test(t);
    if (pendingOriginal && isPureChinese(entry.text) && isPureEnglish(pendingOriginal.original)) {
        return true;
    }
    // 🚀 规则4: 如果本行长度 < 上一行长度 * 0.5 → 可能是翻译（中文比英文短）
    if (pendingOriginal && entry.text.length < pendingOriginal.original.length * 0.5) {
        return true;
    }
    return false;
};
```

```javascript
            return rect.width > 0 && rect.height > 0;
        });
        setFocus(firstVisible >= 0 ? firstVisible : 0);
        return;
    }

    const currentEl = focusableElements[currentFocusIndex];
    const currentRect = currentEl.getBoundingClientRect();
    const curX = currentRect.left + currentRect.width / 2;
    const curY = currentRect.top + currentRect.height / 2;

    let bestTargetIdx = -1;
    let minScore = Infinity;

    focusableElements.forEach((targetEl, idx) => {
        if (idx === currentFocusIndex) return;

        const targetRect = targetEl.getBoundingClientRect();
        
        // 🩹 新增：跳过不可见的元素
        if (targetRect.width === 0 || targetRect.height === 0) return;
        
        const tarX = targetRect.left + targetRect.width / 2;
        const tarY = targetRect.top + targetRect.height / 2;

        const dx = tarX - curX;
        const dy = tarY - curY;

        // 过滤绝对不符合方向要求的元素（比如按"右"时，过滤掉所有在左边的元素）
        let isOpposite = false;
        switch(dir) {
            case 'left': if (dx >= 0) isOpposite = true; break;
            case 'right': if (dx <= 0) isOpposite = true; break;
            case 'up': if (dy >= 0) isOpposite = true; break;
            case 'down': if (dy <= 0) isOpposite = true; break;
        }
        if (isOpposite) return;

        // 🩹 优化：二维欧氏几何加权评分 + 可见性惩罚 + 滚动位置奖励
        let score = 0;
        
        // 基础距离评分
        if (dir === 'left' || dir === 'right') {
            score = Math.abs(dx) + Math.abs(dy) * 2.5;
        } else {
            score = Math.abs(dy) + Math.abs(dx) * 2.5;
        }
        
        // 🩹 新增：可见性惩罚（如果目标元素在视口外，增加惩罚分）
        const viewportHeight = window.innerHeight;
        const viewportWidth = window.innerWidth;
        if (targetRect.top < 0 || targetRect.bottom > viewportHeight ||
            targetRect.left < 0 || targetRect.right > viewportWidth) {
            score += 10000;  // 大惩罚，基本排除视口外元素
        }
        
        // 🩹 新增：元素大小奖励（优先跳到更大的元素，如按钮）
        const area = targetRect.width * targetRect.height;
        score -= Math.log(area) * 0.5;

        if (score < minScore) {
            minScore = score;
            bestTargetIdx = idx;
        }
    });

    if (bestTargetIdx !== -1) {
        setFocus(bestTargetIdx);
    }
};
```

**关键修改点:**
1. **跳过不可见元素** — 防止焦点跳到视口外
2. **可见性惩罚** — 视口外元素评分大幅增加，基本不会被选中
3. **元素大小奖励** — 优先跳到更大的元素（如按钮）

```javascript
// 🩹 v2.8.9: 完善手柄按钮映射，支持所有浮窗完全操作
function pollGamepad() {
    if (!hasGamepad) return;
    
    const gp = navigator.getGamepads()[currentGamepadIndex];
    if (!gp) return;
    
    // 读取当前帧的按钮状态
    const buttons = gp.buttons.map(b => b.pressed || b.value > 0.5);
    const axes = Array.from(gp.axes);
    
    // --- 最高优先级：B 键（button 1）拦截处理 ---
    if (buttons[1] && !prevButtons[1]) {
        handleGlobalClose({ type: 'keydown', key: 'Escape' });
        prevButtons = buttons.slice();
        return;
    }
    
    // --- A 键（button 0）：确认/激活 ---
    if (buttons[0] && !prevButtons[0]) {
        if (currentFocusIndex >= 0 && focusableElements[currentFocusIndex]) {
            const target = focusableElements[currentFocusIndex];
            
            // 🩹 根据元素类型触发不同操作
            if (target.tagName === 'BUTTON' || target.classList.contains('btn-glass')) {
                target.click();  // 按钮 → 点击
            } else if (target.type === 'range') {
                // 滑块 → 进入微调模式（用左右键调整）
                // 微调模式由方向键处理逻辑负责
            } else if (target.type === 'checkbox') {
                target.checked = !target.checked;  // 复选框 → 切换
                target.dispatchEvent(new Event('change'));
            } else if (target.tagName === 'SELECT') {
                target.focus();  // 下拉菜单 → 聚焦并展开
            } else {
                target.click();  // 其他 → 默认点击
            }
        }
    }
    
    // --- 方向键（D-Pad 或左摇杆）：焦点导航 ---
    const deadzone = 0.3;
    const axisX = Math.abs(axes[0]) > deadzone ? axes[0] : 0;
    const axisY = Math.abs(axes[1]) > deadzone ? axes[1] : 0;
    
    if (Math.abs(axisX) > 0.5) {
        moveFocus2D(axisX > 0 ? 'right' : 'left');
        lastNavTime = Date.now();
    }
    if (Math.abs(axisY) > 0.5) {
        moveFocus2D(axisY > 0 ? 'down' : 'up');
        lastNavTime = Date.now();
    }
    
    // --- 肩键 LB/RB（button 4/5）：切换选项卡 / 调整音量 ---
    if (buttons[4] && !prevButtons[4]) {
        // LB → 上一个选项卡（设置浮窗）或减少音量
        if (el.settingsModal.classList.contains('open')) {
            // 设置浮窗中：切换到上一个选项卡
            switchToPrevTab();
        } else {
            adjustVolume(-0.05);
        }
    }
    if (buttons[5] && !prevButtons[5]) {
        // RB → 下一个选项卡（设置浮窗）或增加音量
        if (el.settingsModal.classList.contains('open')) {
            // 设置浮窗中：切换到下一个选项卡
            switchToNextTab();
        } else {
            adjustVolume(0.05);
        }
    }
    
    // --- 触发键 LT/RT（button 6/7）：快进/快退 ---
    if (buttons[6] && !prevButtons[6]) {
        // LT → 快退 5 秒
        audio.currentTime = Math.max(0, audio.currentTime - 5);
        showToast(`⏪ 快退 5秒`);
    }
    if (buttons[7] && !prevButtons[7]) {
        // RT → 快进 5 秒
        audio.currentTime = Math.min(audio.duration, audio.currentTime + 5);
        showToast(`⏩ 快进 5秒`);
    }
    
    // --- X/Y 键（button 2/3）：更多功能 ---
    if (buttons[2] && !prevButtons[2]) {
        // X 键 → 切换播放/暂停（与 A 键不同，X 键全局有效）
        togglePlay();
    }
    if (buttons[3] && !prevButtons[3]) {
        // Y 键 → 切换沉浸模式
        toggleImmersive();
    }
    
    // --- 左摇杆点击（L3，axis 9）：切换焦点模式 ---
    if (buttons[10] && !prevButtons[10]) {
        // 切换焦点模式：正常模式 ↔ 微调模式
        toggleFocusMode();
    }
    
    prevButtons = buttons.slice();
}

// 🩹 辅助函数：切换选项卡（设置浮窗）
function switchToPrevTab() {
    const tabs = document.querySelectorAll('.settings-tab');
    const activeTab = document.querySelector('.settings-tab.active');
    if (!tabs.length || !activeTab) return;
    
    const currentIdx = Array.from(tabs).indexOf(activeTab);
    const prevIdx = (currentIdx - 1 + tabs.length) % tabs.length;
    tabs[prevIdx].click();
}

function switchToNextTab() {
    const tabs = document.querySelectorAll('.settings-tab');
    const activeTab = document.querySelector('.settings-tab.active');
    if (!tabs.length || !activeTab) return;
    
    const currentIdx = Array.from(tabs).indexOf(activeTab);
    const nextIdx = (currentIdx + 1) % tabs.length;
    tabs[nextIdx].click();
}

}
```

```javascript
let sliderFineMode = false;
let currentSlider = null;

function handleSliderGamepad(e) {
    if (!sliderFineMode || !currentSlider) return;
    
    // 左右键调整滑块值
    const step = parseFloat(currentSlider.step) || 1;
    const min = parseFloat(currentSlider.min) || 0;

    if (e.key === 'ArrowLeft' || e.key === 'GamepadLeft') {
        currentSlider.value = Math.max(min, parseFloat(currentSlider.value) - step);
        currentSlider.dispatchEvent(new Event('input'));
    } else if (e.key === 'ArrowRight' || e.key === 'GamepadRight') {
        currentSlider.value = Math.min(max, parseFloat(currentSlider.value) + step);
        currentSlider.dispatchEvent(new Event('input'));
    } else if (e.key === 'Escape' || e.key === 'GamepadB') {
        // 退出微调模式
        sliderFineMode = false;
        currentSlider = null;
        showToast("🎯 退出滑块微调模式");
    }
}
```
