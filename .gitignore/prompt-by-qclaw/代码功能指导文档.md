## §3 关键代码段详解

### 3.1 双语歌词解析 (`app.js` parseLyricText() ~L1370)
```javascript
// v2.8.12 链式+配对解析算法（全球发行增强版）：
// 同时间戳 pair 第一行 = 上一句原文的翻译
// 同时间戳 pair 第二行 = 新原文
// 双语独立空行 → isBreak (verse break，沉浸/PiP 跳过)
// 单语独立空行 → isBlank (有意空置，保留)
// 末尾翻译配对：末三行中若末二/末三同时间戳 → 末行=末二行翻译
// Phase 5b 自启双语：无版权标记但歌词区 pair > single → 自动启用双语模式
// 创作信息检测：40+ 标识基于 TME 清单最终版（中文/英文/多角色合并）
// 名字列表启发式检测：以 '(' 开头、长串 '/'/'&' 分隔名单 → 视为元数据
// 输出: { lyrics, credits, kanaData }

## §4 常见问题定位指南

| 问题领域 | 优先检查文件 | 关键函数 |
|---------|-------------|---------|
| 播放不工作 | `audio-core.js` | `playAudio()`, `audio.ontimeupdate` |
| 歌词显示异常 | `app.js` | `parseLyricText()`, `syncLyrics()` |

## §5 代码修改安全区

### ✅ 可安全修改
- `css/` 下所有文件（纯样式，无副作用）
- `js/utils.js`（工具函数，不影响核心逻辑）
- `index.html` 的 DOM 结构（需同步修改对应 JS 引用）

### ⚠️ 修改需谨慎
- `js/app.js` 的全局变量声明（L37-160）：影响所有模块
- `js/audio-core.js` 的 `playAudio()`：播放逻辑核心
- `js/visualizer.js` 的 `renderVisLoop()`：性能敏感
- `js/ui-core.js` 的 `handleGlobalClose()`：弹窗栈逻辑

### 🔴 禁止修改（除非完全理解）
- `js/app.js` 的 `EnergyMode` 位标志逻辑
- `js/visualizer.js` 的对象池实现（错误会导致内存泄漏）
- `js/storage.js` 的 IndexedDB 版本升级逻辑
