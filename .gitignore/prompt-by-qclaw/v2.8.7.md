## 问题 4：系统性重写音轨交叉/淡入淡出切歌功能

### 📋 问题描述

**用户需求：**  
当前 crossfade 功能可用性极差，甚至不如上一个版本（v2.8.2 之前），连最基本的淡入都做不到。需要基于最新代码（Windows 11 / Android 16 + Chrome 148）系统性地重写整个功能，不考虑兼容性。

**当前行为 (v2.8.3+):**  
- Crossfade 触发后，下一首歌曲无法正常淡入  
- 有时两首歌同时播放  
- 有时下一首歌曲音量突然跳到最大  
- `isFading` 标志管理混乱，导致重复触发

**期望行为：**  
- 当前歌曲平滑淡出（音量从 100% → 0%）  
- 下一首歌曲平滑淡入（音量从 0% → 100%）  
- 交叉期间不会同时播放两首歌  
- 所有边界情况（shuffle、repeat、手动切歌）都正确处理

### 🔍 根因分析

#### 代码位置：`js/app.js` ~L1650-1850

**核心问题 1：`finishCrossfade()` 音频切换逻辑错误**
```javascript
function finishCrossfade(nextAudio, nextIndex, userVolume) {
    // ❌ 错误：只是复制 src，不转移整个音频元素
    audio.src = nextAudio.src;
    audio.volume = userVolume;
    
    // ❌ 错误：nextAudio 已经播放了一段时间，但 audio 从头开始
    audio.play();
    
    nextAudio.pause();
    nextAudio.src = '';  // ❌ 可能导致内存泄漏
}
```

**问题：**
- `audio.src = nextAudio.src` 会让音频**重新加载**，失去淡入效果  
- 应该直接**转移音频元素**（swap `audio` and `nextAudio`）  
- `nextAudio` 没有正确清理（需要 `nextAudio.load()` 重置状态）

---

**核心问题 2：`isFading` 标志管理混乱**
```javascript
function setupCrossfade() {
    const checkCrossfade = () => {
        if (!crossfadeEnabled || isRepeatOne || playlist.length < 2 || isFading) {
            requestAnimationFrame(checkCrossfade);
            return;
        }
        // ...
        if (remaining <= crossfadeDuration && remaining > 0.5) {
            isFading = true;  // ✅ 设置标志
            triggerCrossfade();
        }
        // ...
    };
}

function triggerCrossfade() {
    // ❌ 如果预加载失败，isFading 没有重置
    nextAudio.addEventListener('error', () => {
        console.warn('下一首预加载失败，使用旧版淡入淡出');
        triggerFadeOutLegacy(userVolume);  // ❌ 没有 isFading = false
    }, { once: true });
}

function finishCrossfade() {
    // ❌ isFading 在 audio.play().then() 内部才设为 false
    audio.play().then(() => {
        isFading = false;  // ❌ 如果 play() 失败，标志永远不会重置
    }).catch(e => {
        isFading = false;  // ✅ 只有 catch 里重置了
    });
}
```

**问题：**
- `isFading` 标志在多个地方设置/重置，容易遗漏  
- 如果 `triggerCrossfade()` 失败，`isFading` 永远为 `true`，后续切歌全部失效  
- 应该使用 **状态机** 管理 crossfade 状态，而不是单一 boolean

---

**核心问题 3：Legacy 淡入淡出逻辑粗糙**
```javascript
function triggerFadeInLegacy(targetVolume) {
    audio.volume = 0;
    const step = 0.05;
    const fadeInDuration = 1.5;
    const stepsCount = targetVolume / step;
    const intervalTime = stepsCount > 0 ? (fadeInDuration * 1000) / stepsCount : 100;
    
    let fadeInInterval = null;
    
    const onPlaying = () => {
        audio.removeEventListener('playing', onPlaying);
        
        fadeInInterval = setInterval(() => {
            if (audio.volume < targetVolume - step) {
                audio.volume = Math.min(targetVolume, audio.volume + step);
            } else {
                clearInterval(fadeInInterval);
                audio.volume = targetVolume;
                isFading = false;
            }
        }, intervalTime);
    };
    
    // ❌ 问题：如果音频已经可以播放，直接调用 onPlaying()
    // 但如果音频缓冲不足，会等待 'playing' 事件
    // 这个逻辑不稳定，容易导致淡入失败
}
```

**问题：**
- 使用 `setInterval` + 固定步长，音质差（能听到步进感）  
- 依赖 `playing` 事件，但事件触发时机不稳定  
- 没有使用 `requestAnimationFrame`，导致帧率不匹配

---

### 🔧 修复方案：系统性重写

#### 设计原则

1. **使用 `requestAnimationFrame` 实现平滑动画**  
   - 替代 `setInterval`，帧率稳定 60fps  
   - 使用 `performance.now()` 计算精确时间差

2. **音频元素转移而非复制 `src`**  
   - 创建 `audioPool = [audio, nextAudio, nextAudio2]` 对象池  
   - Crossfade 时直接 swap 活跃音频元素  
   - 避免重新加载导致的延迟和音质损失

3. **状态机管理 Crossfade 生命周期**  
   - `CrossfadeState.IDLE` - 空闲  
   - `CrossfadeState.PRELOADING` - 预加载中  
   - `CrossfadeState.FADING` - 交叉淡入淡出中  
   - `CrossfadeState.COMPLETED` - 完成（等待清理）  
   - 每个状态只能转移到特定下一个状态，避免混乱

4. **支持所有边界情况**  
   - Shuffle 模式：`getNextTrackIndex()` 统一选择逻辑  
   - Repeat One：`isRepeatOne` 检查时跳过 crossfade  
   - 手动切歌：强制取消正在进行的 crossfade  
   - 播放列表 < 2：禁用 crossfade

function setupCrossfade() {
    // 取消之前的检测循环
    if (crossfadeRafId) {
        cancelAnimationFrame(crossfadeRafId);
        crossfadeRafId = null;
    }
    
    // 重置状态

    // 状态检查
    if (crossfadeState !== CrossfadeState.IDLE) return;
    
    crossfadeState = CrossfadeState.PRELOADING;
    
    const userVolume = parseFloat(el.volSlider.value);

#### 步骤 4：实现 `performCrossfade()` 核心函数

**文件：** `js/app.js`  
**位置：** 在 `triggerCrossfadeV2()` 之后

**新代码：**
```javascript
// ✅ v2.8.7: 执行 Crossfade 动画
function performCrossfade(nextAudio, nextIndex, userVolume) {
    const startTime = performance.now();
    const duration = crossfadeDuration * 1000;  // 转换为毫秒
    
    // 开始播放下一首（静音）
    nextAudio.play().catch(e => {
        console.error('Crossfade 播放失败', e);
        crossfadeState = CrossfadeState.IDLE;
        goNext();
        return;
    });
    
    // 动画帧更新函数
    const fadeStep = (timestamp) => {
        // 检查状态（可能被手动切歌取消）
        if (crossfadeState !== CrossfadeState.FADING) return;
        
        const elapsed = timestamp - startTime;
        const progress = Math.min(elapsed / duration, 1);
        
        // 当前歌曲淡出
        audio.volume = userVolume * (1 - progress);
        // 下一首淡入
        nextAudio.volume = userVolume * progress;
        
        if (progress < 1) {
            crossfadeRafId = requestAnimationFrame(fadeStep);
        } else {
            // Crossfade 完成
            finishCrossfadeV2(nextAudio, nextIndex, userVolume);
        }
    };
    
    // 启动动画
    crossfadeRafId = requestAnimationFrame(fadeStep);
}
```

**关键改动：**
1. 使用 `performance.now()` 计算精确时间差  
2. 进度计算 `progress = elapsed / duration`，平滑线性过渡  
3. 每帧更新两个音频的音量，实现真正的"交叉"淡入淡出  
4. 支持外部取消（检查 `crossfadeState`）

### 📐 数学公式：音量曲线优化（可选）

**当前实现：** 线性淡入淡出  
```javascript
volume = userVolume * progress;  // 线性
```

**问题：** 线性曲线听起来不自然（人耳对音量的感知是对数的）

**优化方案：** 使用指数曲线或正弦曲线  
```javascript
// 指数曲线（更自然）
volume = userVolume * Math.pow(progress, 2);

// 正弦曲线（最平滑）
volume = userVolume * Math.sin(progress * Math.PI / 2);
```

**修改位置：** `performCrossfade()` 函数中的 `fadeStep`  
```javascript
const fadeStep = (timestamp) => {
    const elapsed = timestamp - startTime;
    const progress = Math.min(elapsed / duration, 1);
    
    // 当前歌曲淡出（指数曲线）
    audio.volume = userVolume * Math.pow(1 - progress, 2);
    // 下一首淡入（指数曲线）
    nextAudio.volume = userVolume * Math.pow(progress, 2);
    
    // ...
};
```

**注意：** 这个优化是可选的，线性曲线已经比之前的所有实现都好。指数/正弦曲线需要用户反馈调整参数。
