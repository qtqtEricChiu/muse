## 问题 1：双语 LRC 匹配逻辑错误（高优先级）

### 现象
同时间戳的双语歌词（原文+翻译）在 `parsedLyrics` 数组中被拆分为两个独立条目，导致：
- 歌词面板显示重复时间戳行
- 沉浸模式当前行/下一行判断错乱
- 进度条悬停预览显示不完整

### 根因分析

现有 `parseLyricText` 函数（约第 1260~1279 行）的合并逻辑：

```javascript
// 现有问题代码（示意）
if (prev && Math.abs(prev.time - time) < 0.02) {
    // 合并到前一个条目
    if (!prev.original) prev.original = prev.text;
    prev.translation = text;
    prev.isBilingual = true;
    prev.text = prev.original + '\n' + text; // 合并显示文本
}
```

**问题：**
1. 合并后 `prev.text` 被覆盖为合并文本，但 `syncLyrics` 中仍使用 `text` 字段做比较判断
2. `getLyricAtTime` 返回的 `text` 字段格式不一致（有时是原文，有时是合并文本）
3. 沉浸模式判断 `isBilingual` 时，下一行查找逻辑跳过同时间戳条目的方式不可靠

### 修复方案

#### 步骤 1：重构 `parseLyricText` 合并算法

```javascript
function parseLyricText(text) {
    const lines = text.split(/\r?\n/);
    const rawEntries = [];
    
    // 第一步：提取所有带时间戳的条目
    lines.forEach(line => {
        const match = line.match(/\[(\d{2}):(\d{2}\.\d{2,3})\](.*)/);
        if (match) {
            const time = parseInt(match[1]) * 60 + parseFloat(match[2]);
            const text = match[3].trim();
            if (text) {
                rawEntries.push({ time, text });
            }
        }
    });
    
    // 第二步：按时间戳分组合并
    const merged = [];
    for (let i = 0; i < rawEntries.length; i++) {
        const curr = rawEntries[i];
        const prev = merged[merged.length - 1];
        
        // 判断是否为同时间戳（容差 0.02 秒）
        if (prev && Math.abs(prev.time - curr.time) < 0.02) {
            // 同时间戳：第一个为原文，第二个为翻译
            if (!prev.isBilingual) {
                prev.original = prev.text;
                prev.translation = curr.text;
                prev.isBilingual = true;
                // 保留原始 text 用于单语场景兼容
                prev.text = prev.original;
            }
            // 忽略第三个及以上同时间戳条目（异常情况）
        } else {
            merged.push({
                time: curr.time,
                text: curr.text,
                original: curr.text,
                translation: null,
                isBilingual: false
            });
        }
    }
    
    return merged;
}
```

#### 步骤 3：修复 `getLyricAtTime` 进度条预览

```javascript
function getLyricAtTime(time) {
    if (!parsedLyrics.length) return null;
    const adjustedTime = time - lyricsOffset;
    let best = null;
    for (let i = 0; i < parsedLyrics.length; i++) {
        if (parsedLyrics[i].time <= adjustedTime) best = parsedLyrics[i];
        else break;
    }
    
    // 🚀 v2.8.4: 双语歌词显示原文+翻译
    if (best && best.isBilingual) {
        return {
            ...best,
            // 返回合并文本用于预览显示
            text: `${best.original} | ${best.translation}`
        };
    }
    return best;
}
```

## 问题 2：沉浸模式歌词动画频闪（高优先级）

### 现象
沉浸模式下歌词切换时出现快速闪烁、透明度抖动，视觉上不连贯。

### 根因分析

1. **硬编码 setTimeout 延迟**：现有代码使用固定 200ms setTimeout，与 CSS transition 不同步
2. **opacity 直接赋值**：`el.style.opacity = 0` 后立即 `= 1`，没有利用 CSS transition
3. **频繁 DOM 操作**：每次 `syncLyrics` 调用都进行 innerHTML 替换，即使内容未变
4. **缺少过渡状态锁**：多个 setTimeout 可能同时执行，导致动画堆叠冲突

### 修复方案

#### 方案：引入 CSS transition + 状态锁

**1. CSS 层（`css/immersive.css`）**

```css
.imm-curr-line, .imm-next-line {
    transition: opacity 0.2s ease-out, transform 0.2s ease-out;
    will-change: opacity, transform;
}

.imm-curr-line.switching, .imm-next-line.switching {
    opacity: 0;
    transform: translateY(8px);
}

.imm-curr-line.active, .imm-next-line.active {
    opacity: 1;
    transform: translateY(0);
}
```

**2. JS 层（`js/app.js` `syncLyrics` 函数）**

```javascript
// 在文件顶部添加过渡锁
let lrcTransitionLock = false;

const syncLyrics = (force = false) => {
    if (!parsedLyrics.length) return;
    const cur = audio.currentTime - lyricsOffset;
    let activeIdx = -1;
    for (let i = 0; i < parsedLyrics.length; i++) {
        if (cur >= parsedLyrics[i].time - 0.2) activeIdx = i;
        else break;
    }

    // 主界面歌词面板（保持现有逻辑）
    // ...

    // 🚀 v2.8.4: 沉浸模式歌词显示 - 防频闪重构
    if (activeIdx !== -1 && !el.immLrcCenter.classList.contains('hidden')) {
        const currentLrc = parsedLyrics[activeIdx];
        
        // 如果正在过渡中且不是强制刷新，跳过本次更新
        if (lrcTransitionLock && !force) return;
        
        const updateImmLine = (element, newHtml, isSwitching) => {
            // 内容未变，跳过
            if (element.dataset.lastHtml === newHtml && !force) return;
            
            lrcTransitionLock = true;
            element.dataset.lastHtml = newHtml;
            
            // 添加退出动画类
            element.classList.add('switching');
            element.classList.remove('active');
            
            // 监听 transitionend 事件
            const onTransitionEnd = (e) => {
                if (e.propertyName !== 'opacity') return;
                
                element.innerHTML = newHtml;
                element.classList.remove('switching');
                element.classList.add('active');
                
                // 清理监听器
                element.removeEventListener('transitionend', onTransitionEnd);
                
                // 延迟释放锁，确保进入动画完成
                setTimeout(() => {
                    lrcTransitionLock = false;
                }, 250);
            };
            
            element.addEventListener('transitionend', onTransitionEnd);
            
            // 兜底：如果 transition 未触发（如元素隐藏），强制完成
            setTimeout(() => {
                if (lrcTransitionLock) {
                    element.innerHTML = newHtml;
                    element.classList.remove('switching');
                    element.classList.add('active');
                    lrcTransitionLock = false;
                }
            }, 300);
        };
        
        if (currentLrc.isBilingual) {
            const html = `<span class="imm-original">${escapeHtml(currentLrc.original)}</span>` +
                        `<span class="imm-translation">${escapeHtml(currentLrc.translation)}</span>`;
            updateImmLine(el.immCurrLine, html, true);
            
            // 下一行
            let nextIdx = activeIdx + 1;
            while (nextIdx < parsedLyrics.length && 
                   Math.abs(parsedLyrics[nextIdx].time - currentLrc.time) < 0.02) {
                nextIdx++;
            }
            const nextLrc = nextIdx < parsedLyrics.length ? parsedLyrics[nextIdx] : null;
            const nextHtml = nextLrc ? escapeHtml(nextLrc.text) : '';
            updateImmLine(el.immNextLine, nextHtml, false);
        } else {
            updateImmLine(el.immCurrLine, escapeHtml(currentLrc.text), true);
            const nextHtml = activeIdx + 1 < parsedLyrics.length ? 
                escapeHtml(parsedLyrics[activeIdx + 1].text) : '';
            updateImmLine(el.immNextLine, nextHtml, false);
        }
    }
};
```

## 问题 3：一键节能失效且覆盖顺序错误（中优先级）

### 现象
1. 开启一键节能后再打开画中画，画中画关闭后一键节能状态丢失
2. 用户手动开启一键节能后，画中画临时节能覆盖导致状态混乱
3. `visibilitychange` 事件在画中画运行时错误退出节能模式

### 根因分析

**状态机设计缺陷：**

```javascript
// 现有变量（约第 92~97 行）
let isEnergySaving = false;        // 全局节能状态
let oneClickEnergySaving = false;  // 一键节能开关
let frameEnergySaving = false;     // 帧率节能开关
let pipTempEnergySaving = false;   // PiP 临时节能标记
```

**问题：**
1. `enterEnergySaving('pip')` 和 `enterEnergySaving('oneclick')` 都设置 `isEnergySaving = true`，没有区分来源
2. `exitEnergySaving()` 直接清除 `isEnergySaving = false`，不检查是否还有其他节能需求
3. `visibilitychange` 的 `exitEnergySaving()` 调用没有检查 `oneClickEnergySaving`

### 修复方案

#### 步骤 1：重构节能状态机

```javascript
// 🚀 v2.8.4: 节能模式状态机重构
const EnergyMode = {
    NONE: 0,
    ONE_CLICK: 1,    // 用户手动开启
    PIP_TEMP: 2,     // 画中画临时节能
    FRAME_LIMIT: 4,  // 帧率限制（30fps）
    VISIBILITY: 8    // 标签页隐藏
};

let energyModeFlags = EnergyMode.NONE;

// 检查是否应该处于节能状态
function shouldBeEnergySaving() {
    return energyModeFlags !== EnergyMode.NONE;
}

// 进入指定节能模式（位运算，支持叠加）
function enterEnergySaving(mode) {
    const wasSaving = shouldBeEnergySaving();
    energyModeFlags |= mode;
    
    if (!wasSaving && shouldBeEnergySaving()) {
        // 首次进入节能状态，执行实际节能操作
        applyEnergySaving(true, mode);
    }
    
    // 更新 CSS 类
    const wrapper = document.querySelector('.player-wrapper');
    if (wrapper) wrapper.classList.toggle('pip-standby', 
        (energyModeFlags & (EnergyMode.PIP_TEMP | EnergyMode.VISIBILITY)) !== 0);
}

// 退出指定节能模式
function exitEnergySaving(mode = EnergyMode.PIP_TEMP) {
    const wasSaving = shouldBeEnergySaving();
    energyModeFlags &= ~mode;
    
    if (wasSaving && !shouldBeEnergySaving()) {
        // 完全退出节能状态
        applyEnergySaving(false);
    }
    
    // 如果还有一键节能，保持节能效果但移除 pip-standby CSS
    const wrapper = document.querySelector('.player-wrapper');
    if (wrapper) {
        const hasPipStandby = (energyModeFlags & (EnergyMode.PIP_TEMP | EnergyMode.VISIBILITY)) !== 0;
        wrapper.classList.toggle('pip-standby', hasPipStandby);
    }
}

// 实际应用/取消节能效果
function applyEnergySaving(enable, triggerMode) {
    isEnergySaving = enable;
    
    if (enable) {
        // 清空粒子池
        particles.length = 0;
        ripples.length = 0;
        flowField = [];
        
        // 清空 Canvas
        const bgCtx = el.bgColor.getContext('2d');
        if (bgCtx) bgCtx.clearRect(0, 0, el.bgColor.width, el.bgColor.height);
        
        if (spectrumCtxMain) {
            const cvs = el.canvasMain;
            if (cvs) spectrumCtxMain.clearRect(0, 0, cvs.width, cvs.height);
        }
        
        // 降低歌词同步频率
        if (lrcTimer) clearInterval(lrcTimer);
        lrcTimer = setInterval(() => syncLyrics(true), 500);
        
        // 一键节能不添加 pip-standby（保持正常亮度）
        if (triggerMode === EnergyMode.ONE_CLICK) {
            showToast("🔋 一键节能已开启", "⚡");
        }
    } else {
        // 恢复歌词高频同步
        if (lrcTimer) { clearInterval(lrcTimer); lrcTimer = null; }
        
        // 恢复视觉特效
        if (analyser && !isImmersiveMode) {
            requestAnimationFrame(renderVisLoop);
        }
        
        showToast("🔋 节能模式已退出", "⚡");
    }
}
```

## 问题 4：曲库浮窗在播放专辑后无法打开（中优先级）

### 现象
1. 从曲库浮窗打开专辑详情，点击"播放整张专辑"
2. 专辑开始播放，曲库浮窗和专辑详情浮窗都关闭
3. 再次点击曲库按钮，浮窗无法打开或显示空白

### 根因分析

**问题代码路径（`showAlbumDetail` 函数）：**

```javascript

    detailModal.remove();
    parentModal.remove();  // 🔴 问题：直接移除 DOM，未使用 closeAllModals
```

**问题：**
1. `parentModal.remove()` 直接从 DOM 移除，但曲库浮窗的 `open` 类未清除

3. `playlist` 被替换后，`renderPlaylist` 可能引用旧数据

## 问题 5：手柄 B 键退出浮窗失效且需最高优先级（高优先级）

### 现象
1. 手柄 B 键（按钮索引 1）无法关闭浮窗
2. 在专辑详情浮窗中按 B 键无响应
3. B 键在沉浸模式中应先退出沉浸，而非直接关闭浮窗

### 根因分析

**现有代码（约第 4354~4360 行）：**

```javascript
if (btns[1] && !prevPadBtns[1]) {
    const closed = handleGlobalClose();
    if (!closed && isImmersiveMode) {
        toggleImmersiveMode();
    }
}
```

**问题：**
1. `handleGlobalClose()` 的 LIFO 逻辑对动态创建的浮窗（专辑详情）处理不正确
2. 动态浮窗关闭后未更新焦点上下文
3. B 键在沉浸模式中的优先级：应先退出沉浸，再处理浮窗

### 修复方案

#### 步骤 1：重构 `handleGlobalClose` 函数

```javascript
// 🚀 v2.8.4: 统一弹窗栈关闭管理器（增强版）
function handleGlobalClose() {
    // 🔧 修复：扫描所有浮窗，包括动态创建的
    const allModals = Array.from(document.querySelectorAll('.modal-overlay'));
    const activeModals = allModals.filter(m => {
        // 检查是否处于打开状态
        const isOpen = m.classList.contains('open');
        const isVisible = m.style.display !== 'none' && m.style.visibility !== 'hidden';
        const inDom = document.body.contains(m);
        return (isOpen || isVisible) && inDom;
    });
    
    if (activeModals.length === 0) return false;
    
    // 按 z-index 排序，关闭最上层的
    activeModals.sort((a, b) => {
        const zA = parseInt(getComputedStyle(a).zIndex) || 0;
        const zB = parseInt(getComputedStyle(b).zIndex) || 0;
        return zB - zA;
    });
    
    const topModal = activeModals[0];
    
    // 区分静态和动态弹窗
    const staticIds = ['playlistModal', 'settingsModal', 'fileInfoModal', 'helpModal', 'coverLibraryModal'];
    
    if (staticIds.includes(topModal.id)) {
        // 静态弹窗：移除 open 类
        topModal.classList.remove('open');
    } else {
        // 动态弹窗：触发退出动画后移除
        topModal.classList.remove('open');
        setTimeout(() => {
            if (topModal.parentNode) {
                topModal.remove();
            }
            updateFocusContext();
        }, 400);
    }
    
    // 更新焦点
    setTimeout(() => updateFocusContext(), 50);
    
    return true;
}
```

#### 步骤 2：重构手柄 B 键逻辑

```javascript

    // 🔧 v2.8.4: B 键优先级：浮窗 > 沉浸模式 > 无操作
```

#### 步骤 3：增强 `updateFocusContext` 函数

```javascript
const updateFocusContext = () => {
    focusableElements.forEach(el => el.classList.remove('gamepad-focus'));
    currentFocusIndex = -1;
    
    // 🔧 v2.8.4: 按 z-index 优先级检测浮窗
    const allModals = Array.from(document.querySelectorAll('.modal-overlay'));
    const activeModals = allModals.filter(m => {
        return m.classList.contains('open') && document.body.contains(m);
    }).sort((a, b) => {
        const zA = parseInt(getComputedStyle(a).zIndex) || 0;
        const zB = parseInt(getComputedStyle(b).zIndex) || 0;
        return zB - zA;
    });
    
    // 获取最上层浮窗
    const topModal = activeModals[0];
    
    if (topModal) {
        // 优先聚焦最上层浮窗内的元素
        focusableElements = Array.from(topModal.querySelectorAll('.focusable'));
        
        // 如果没有 focusable 元素，尝试聚焦弹窗本身
        if (focusableElements.length === 0 && topModal.querySelector('.modal-content')) {
            focusableElements = [topModal.querySelector('.modal-content')];
        }
    } else if (isImmersiveMode) {
        focusableElements = Array.from(el.viewImm.querySelectorAll('.focusable'));
    } else {
        focusableElements = Array.from(el.viewMain.querySelectorAll('.focusable'));
    }
    
    // 自动聚焦第一个元素
    if (focusableElements.length > 0) {
        setFocus(0);
    }
};
```
