# v2.9.0 - 多角色创作信息格式支持修复指导

## 📋 文档概述

**问题描述**：TME LRC 文件中，创作信息行可能包含**多角色合并格式**（如 `制作人/作曲/编曲：Sihan`），现有代码不支持此格式，导致第一行创作信息未被识别为元数据，后续所有创作信息提取失败。

**修复目标**：增强创作信息检测逻辑，支持多角色合并格式（用 `/` 分隔）。

**适用版本**：MBolka Player v2.9.0+  
**文件位置**：`Q:\数据\Web\code\muse\prompt-by-qclaw\v2.9.0_多角色创作信息格式支持.md`  
**创建时间**：2026-06-04 00:53 GMT+8

## 🔍 问题分析

### 问题格式示例

**文件**：`DAMIDAMI-《绝区零》卢西娅EP - Sihan&三Z-STUDIO&HOYO-MiX.lrc`

```
[00:00.00]DAMIDAMI - Sihan/三Z-STUDIO/HOYO-MiX
[00:00.64]
[00:00.64]制作人/作曲/编曲：Sihan              ← 多角色合并格式
[00:01.66]
[00:01.66]作词：Lyla P./Sihan/Nellie Fors/SmileL/Nanyan P/Zilan Li
[00:02.87]
[00:02.87]演唱：Aylin Abbasova
...
```

### 根本原因

**现有代码**（app.js 约1435行）：
```javascript
const CREDIT_PAT = /^(词|曲|编曲|混音|录音|制作|制作人|制作\/版权|吉他|贝斯|键盘|鼓|和声|弦乐|配唱|出品|发行|母版|OP|SP|ISRC|演唱|Remixed|Keyboard|Synthesizer|Bass|Drum|Background|Vocal|Digital|Recording|Mix|All instruments|Drum Programming|Vocal Arrangement|Digital Editing|Recording Engineers|Mix Engineer)[：:\s]/i;
```

**问题**：
1. 正则表达式**只匹配以单个角色开头的行**
2. **不支持** `角色1/角色2/角色3：` 这种多角色合并格式
3. 第一行 `制作人/作曲/编曲：Sihan` 未被识别为元数据
4. `lyricStart` 被错误设置为 0 或很小的值
5. 后续所有创作信息虽然能被识别，但已经被排除在元数据区域外
