## 修复：MBolka Player 按钮不可点击问题

### 修复的文件

| 文件 | 问题 | 修复 |
|------|------|------|
| `js/ui-core.js` | `const toggleImmersiveMode` 在 line 776 定义，但 line 266/267/285 已引用 → TDZ ReferenceError，导致 line 268~801 整个死亡 | 改为 `function` 声明（函数提升） |
| `js/ui-core.js` | `const toggleDarkMode` 在 line 763 定义，但 line 341 已引用 → TDZ ReferenceError | 改为 `function` 声明 |
| `js/ui-core.js` | `const toggleColorMode` 在 line 335 后定义，但 line 340 已引用 → TDZ | 改为 `function` 声明 |
| `js/ui-core.js` | `toggleFullscreen` 箭头函数缺 `};` | 补充闭合括号 |
| `js/app.js` | `window.MBolka = {...}` 后多了一个 `})` | 删除多余 `})` |
| `js/globals.js` | `el` 初始化依赖 `DOMContentLoaded` 事件，但脚本在 body 末尾已可安全初始化 | 立即初始化 `el` 对象 |

### 根因
模块化时把顶层事件绑定代码放到了文件头部（line 266~341），但 `const` 函数声明还在原地（line 763~800），由于 `const` 有 TDZ，执行到绑定代码时函数未定义 → ReferenceError → 脚本中断。

## gamepad.js 手柄状态机修复（2026-07-05）

### 修改内容（`js/gamepad.js`）
1. **长按触发检查**（在 `const btns = ...` 之后）：遍历 `_btnTimers`，按钮持续按下超过 `LONG_PRESS_MS(800ms)` 时触发对应功能
   - A(0)=右键菜单, X(2)=收藏, Y(3)=深色模式, LB(4)=睡眠定时, RB(5)=统计面板, LT(6)=睡眠定时, RT(7)=统计面板, View(8)=曲库, Start(9)=播放列表
2. **组合键检测**：View+Start=PiP, View+RB=帮助面板（使用 `btns` 替代 `buttons`）
3. **长按/双击状态追踪**：遍历所有按键，按下时记录 `_btnTimers[i]`，释放时检查短按/双击
   - X 按钮(2) 300ms 内双击 = 切换播放模式
4. **Seek 模式**：LT+RT 双扳机同时按住进入，左摇杆左右控制进度（>0.7=5秒, 其他=1秒），松开退出
